Organizational Success Factors

Does your IT organization widely recognize IT as a competitive advantage to your business?

Can individuals contribute to any technical project within your organization?

Are there clearly defined business and technical champions to lead change?

Has a project with measureable business value been defined to be automatically deployed to production?

Is risk taking encouraged and are failures learned from in a positive way?

Do you have quantifiable goals visible to all?

Local Development

Do developers run Chef local testing tools?

Can developers download code dependancies in a friction-free manner?

Do committed code changes have a reasonable expectation of working in CK and production?

Code Collaboration

Can any potential contributer to a project find the code and documentation with minimal assistance?

Are all changes seen by at least two people who have relevant skill and contextual knowledge?

Continuous Integration

Does all code pass through a CI tool before being deployed?

Are all deployments, from pre-prod to production, governed by pipelines?

Does your CI tool make the quality of your code base highly visible?

Is your codebase always in a releasable state?

Chef Code Deployment

Are all cookbooks delivered by a pipeline?

Do you promote a consistant set of cookbook versions between environments?

Can you deploy cookbooks without affecting your users?

Application Deployment

Do you have "1-click" application deployment?

Does your pre-production testing give you confidence in your production deployment?

Can you deploy code without affecting your users?

Virtualization as a Service

Do you use code to provision servers?

Do you build your system images from code?

Full-Stack Automation

Do you use code to provision storage and networking resources?

Do you validate all of the infrastructure dependancies during deployment?

Do you have "1-click" service deployment?

Continuous Deployment

Are the developers who wrote a piece of code responsible for its operation in production?

Do you enforce age limits for all deployed systems and infrastructure components?

Do you have pipelines that safely release changes to production without human verification?

Global State

Set data-state="something" on a slide and "something" will be added as a class to the document element when the slide is open. This lets you apply broader style changes, like switching the page background.

State Events

Additionally custom events can be triggered on a per slide basis by binding to the data-state name. 


Reveal.addEventListener( 'customevent', function() {
	console.log( '"customevent" has fired' );
} );

Take a Moment

Press B or . on your keyboard to pause the presentation. This is helpful when you're on stage and want to take distracting slides off the screen.
